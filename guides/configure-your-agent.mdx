After you [create an agent](/guides/create-an-agent), Intrlume drops you into the **workflow editor** — a visual map of the conversation. This is where you refine what the agent says, the steps it moves through, and how it decides where to go next. Open it anytime from **Agents → your agent**.

<Image src="https://app.intrlume.tech/docs/images/workflow-editor.png" alt="The Intrlume workflow editor — the conversation graph and the Node Properties panel" width="1600" height="917" />

## The conversation flow

Every agent follows the same shape: a **system** layer, a **start** node, one or more **main task** nodes, and **end** nodes.

| Piece | On the canvas | What it does |
| --- | --- | --- |
| **System** | *Global Context* (dashed box) | The agent's identity, persona, and always-on rules. Assembled for you; editable if you need to override it. |
| **Start** | *Greeting* | Speaks the generated greeting, handles the caller's first reply, then routes to the main flow. |
| **Main task** | e.g. *Lead Qualification* | The working steps of the conversation — qualify, book, collect, answer. |
| **End** | e.g. *End — Qualified Lead* | Terminal outcomes. Each end node maps to a **disposition** you'll see in call logs. |

<Callout kind="info">
  **Routing happens through functions, not lines.** Each connection is backed by a *function* the agent can call. The lines on the canvas are just a picture of those functions — edit routing in a node's **Functions** tab, not by drawing arrows.
</Callout>

## Edit a node

Click any node to open the **Node Properties** panel on the right. It has four tabs.

### Prompt

The **Prompt** tab holds the node's **Task Message** — the field that shapes what the agent does at *this* step. Click **expand** for a full-screen editor, and type `@` to insert a [variable](#variables).

**Put the conversation here.** The Task Message is where your instructions about the **flow and steps** belong — the step's objective, the questions to ask and in what order, how to handle answers, and when to move on (for example, *"Qualify the prospect by determining who the training is for, then route to booking."*).

**Don't repeat the agent's identity or style.** Its name, role, behavior, tone, and personality are already generated and live in the **system prompt** — see [What the system prompt already covers](#what-the-system-prompt-already-covers). Restating them in a Task Message is redundant and can conflict with the system prompt.

**Don't put the greeting here.** The opening line is generated and lives on the **start node's Actions tab** as a **Say** action — not in the Task Message. To change the greeting, edit that action (see [Actions](#actions)).

**The start node is special.** Its Task Message should cover only the **opening exchange** — how to respond to the caller's *first* message, and then hand off to the main flow. The greeting is already spoken by the Say action, so the start node simply reacts to the caller's reply and **routes** to a main task node. Keep the actual conversation steps in the main task node(s), not here.

The **Global Context** node is different: its prompt is labeled **System Prompt (auto-assembled)** and is read-only, built from your agent's settings. You can override it from its expand view, but only if you know what you're changing.

<Callout kind="tip">
  Want to sharpen a node's prompt? Paste it into a general-purpose AI tool like **Claude** or **Gemini**, refine the wording there, and bring it back — a fast way to tighten the objective and instructions.
</Callout>

### Functions

Functions are how the conversation moves. **Each function becomes an LLM tool call** the agent can invoke when the moment is right.

| Field | What it means |
| --- | --- |
| **Name** (snake_case) | The function name, e.g. `route_to_main` |
| **Description (when to call)** | Tells the agent *when* to use it, e.g. "Route the caller to the main flow to explore opportunities." |
| **Route to** | The target node — or a **decision** with conditions for branching |
| **Properties (data to collect)** | Inputs the agent gathers before moving on, e.g. `intent` (string, required) |

### Actions

Things the node does automatically, separate from what the LLM says:

- **Pre-actions** run on entering the node — most often **Say** a fixed line (e.g. the greeting).
- **Post-actions** run on leaving — most often **End** the conversation (used on end nodes).

### Settings

- **Context strategy** — *Append* keeps the full conversation history, *Reset* starts the node with a fresh context.
- **Respond immediately** — whether the agent speaks as soon as it enters the node or waits for the caller.

## What the system prompt already covers

Before every call, Intrlume assembles your agent's **system prompt** (the **Global Context**) from its settings. Knowing what's already in there tells you what *not* to repeat in a node's Task Message. In order, it covers:

| Section | What it already handles |
| --- | --- |
| **Identity & personality** | Who the agent is — its name, your company, its role and goal — plus human-like habits (one question at a time, never sounds like a bot). |
| **Environment** | The call context — a live phone call, outbound vs inbound, the current date and time, and anything already known about the caller. |
| **Tone** | How it speaks — the current language, your chosen tone and style, native-script rules for Indian languages, and how to say numbers, dates, and money for your locale. |
| **Tools** | Rules for using tools — calling them silently, looking things up in a linked knowledge base, and ending the call cleanly. |
| **Response guidelines** | How it replies — keep it short and speakable, read back key details like phone numbers and names, collect the values it needs, and stay on goal. |
| **Guardrails** | Its boundaries — stick to facts, decline out-of-scope or unsafe requests, and stay professional with difficult callers. |

Because all of this is already generated, your node **Task Messages** should focus only on the conversation itself — the steps, questions, and routing for each stage.

## Prompting tips

Node prompts drive a **spoken** conversation, which has different rules than writing for chat. A few principles that lift results:

- **Keep replies short** — one or two sentences per turn; long answers become a monologue the caller forgets.
- **One question at a time** — ask, confirm the answer, then move on.
- **Write for the ear** — no bold, lists, or links (the agent reads it aloud); use connectors like "first… then… finally…".
- **Spell out numbers, dates, and money** the way they're spoken (e.g. "January second", "two hundred rupees").
- **Give a fallback for uncertainty** — tell the agent what to say when it doesn't know, so it won't guess.
- **Be specific**, and skip long "never say…" banlists — a short positive instruction works better.
- **Test on real calls and iterate** — judge changes across several calls, not one, and listen end to end.

<Callout kind="info">
  Deeper references: [Vapi's voice prompting guide](https://docs.vapi.ai/prompting-guide) and the general [Prompt Engineering Guide](https://www.promptingguide.ai/). Your agent's **system prompt is generated for you** — these tips are for refining the node-level instructions.
</Callout>

## Variables

Use `{variables}` to personalize what the agent says. Open the left panel to manage them:

- **Variables** — values you set yourself (e.g. a campaign name).
- **Customer Vars** — fields that come from your audience (e.g. `cust_name`, `cust_phone`, and any custom columns).

Type `@` in any prompt to insert a variable token. At call time, Intrlume fills it in per contact — so "Hi {cust_name}" is personalized on every call.

## Knowledge & tools

- **Knowledge Base** (left panel) — link documents so the agent can answer from your content during the call. See [Knowledge Base](/concepts/knowledge-base).
- **Tools** — actions the agent can take mid-call (look up a record, book a slot). Attach them to the node that should use them, and pre-fill any fixed inputs.

## Voice, language & behavior

Click the **agent card** (top-left, with the name and language) to adjust:

- **Voice & pace** — the synthesized voice and how fast it speaks.
- **Language** — a primary language plus an optional second language it can switch to.
- **Background sound** — optional ambient sound (e.g. an office) so calls feel more natural.
- **Locale** — country, currency, and timezone for this agent (how dates and money are spoken, and scheduling).

The **Lock language** toggle (left panel) stops the agent from switching languages mid-call.

## Test it

**Test Call** — try a phone agent live; for web agents, use the web embed to test in the browser.

<Callout kind="tip">
  Make one change, test, and check the transcript in **Call Logs** before the next change — small iterations are easier to debug than big rewrites.
</Callout>
